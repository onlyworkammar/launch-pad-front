## Common UI Components

### 1. Top Navigation Bar
- Present on all authenticated screens
- Contains: Menu toggle, Logo/App name, Search bar, User menu, Logout

### 2. Status Badges
- ACTIVE: Green badge
- INACTIVE: Gray badge
- LOW STOCK: Red/Orange warning badge

### 3. Confidence Indicators (Chat)
- HIGH: Green badge
- MEDIUM: Yellow badge
- LOW: Red badge

### 4. Action Buttons
- Primary: Solid, prominent (Save, Login, Send)
- Secondary: Outlined (Cancel, Clear)
- Danger: Red (Delete)

### 5. Form Validation
- Required fields marked with *
- Inline error messages
- Success indicators

### 6. Loading States
- Spinner for async operations
- Skeleton loaders for data fetching
- Disabled buttons during submission

### 7. Notifications/Toasts
- Success messages (green)
- Error messages (red)
- Info messages (blue)
- Warning messages (yellow)

---

## Responsive Breakpoints

- **Mobile**: < 768px
  - Stack components vertically
  - Collapsible sidebar
  - Full-width cards

- **Tablet**: 768px - 1024px
  - 2-column layouts where appropriate
  - Sidebar can be collapsible

- **Desktop**: > 1024px
  - Full layout as shown in wireframes
  - Sidebar always visible

---

## Color Scheme Suggestions

- **Primary**: Blue (#2563EB) - for primary actions, links
- **Secondary**: Gray (#6B7280) - for secondary elements
- **Success**: Green (#10B981) - for success states, ACTIVE status
- **Warning**: Yellow/Orange (#F59E0B) - for warnings, MEDIUM confidence
- **Error**: Red (#EF4444) - for errors, LOW confidence, LOW STOCK
- **Background**: Light gray (#F9FAFB) - for page backgrounds
- **Card Background**: White (#FFFFFF) - for content cards

---

## Typography

- **Headings**: Bold, larger font sizes
- **Body**: Regular weight, readable size
- **Labels**: Medium weight, slightly smaller
- **Code/JSON**: Monospace font

---

## Accessibility Considerations

- All interactive elements should be keyboard accessible
- Form labels properly associated with inputs
- ARIA labels for icons and buttons
- Color contrast ratios meet WCAG AA standards
- Focus indicators visible on all focusable elements

---

## Notes for Implementation

1. **Authentication Flow**: Redirect to login if token expires (401 response)
2. **Error Handling**: Display user-friendly error messages from API responses
3. **Loading States**: Show loading indicators during API calls
4. **Form Validation**: Client-side validation before API submission
5. **Token Management**: Store JWT in localStorage, include in Authorization header
6. **Pagination**: Implement for large lists (Components, Inventory items)
7. **Search**: Debounce search input for better performance
8. **Caching**: Consider caching frequently accessed data (components list)
